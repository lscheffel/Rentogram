Arquitetura do Sistema

O Rentogram adota uma arquitetura cliente-servidor tradicional, com separação clara de responsabilidades entre frontend, backend e banco de dados.

Visão Geral do Sistema

CORS, Validation]

Componentes Principais

Frontend

  • Tecnologia: React com TypeScript
  • Responsabilidades: Interface de usuário, roteamento, comunicação com API
  • Componentes: Navbar, páginas (Home, Properties, Reservations), formulários

Backend

  • Tecnologia: Node.js com Express
  • Arquitetura: MVC (Model-View-Controller)
  • Componentes:
    • Controllers: Lógica de negócio para propriedades e reservas
    • Models: Interação com banco de dados
    • Routes: Definição de endpoints da API
    • Middlewares: CORS, validação, tratamento de erros

Banco de Dados

  • Tecnologia: SQLite
  • Tabelas: properties, reservations
  • Relação: reservations.property_id → properties.id

Fluxo de Dados

  1. O frontend (React) envia requisições HTTP para o backend via fetch.
  2. O backend (Express) processa as requisições através de middlewares de validação, controladores e modelos.
  3. Os modelos interagem com o banco de dados SQLite para operações CRUD.
  4. Respostas são retornadas ao frontend em formato JSON.

Princípios de Design

  • Separação de Responsabilidades: Cada camada tem responsabilidades bem definidas.
  • API RESTful: Endpoints padronizados para operações CRUD.
  • Validação de Dados: Middleware de validação para garantir integridade.
  • Tratamento de Erros: Middleware global para tratamento consistente de erros.
