Modelos de Dados

O sistema utiliza SQLite como banco de dados relacional, com duas tabelas principais: properties e reservations.

Esquema do Banco de Dados

Tabela properties

Campo Tipo Descrição Restrições
id INTEGER Chave primária auto-incrementada PRIMARY KEY, AUTOINCREMENT
title TEXT Título da propriedade NOT NULL
description TEXT Descrição detalhada
address TEXT Endereço completo NOT NULL
price_per_night REAL Preço por noite NOT NULL
bedrooms INTEGER Número de quartos
bathrooms INTEGER Número de banheiros
max_guests INTEGER Máximo de hóspedes
amenities TEXT Comodidades (string separada por vírgulas)
image_url TEXT URL da imagem principal
created_at TIMESTAMP Data de criação DEFAULT CURRENT_TIMESTAMP
updated_at TIMESTAMP Data de atualização DEFAULT CURRENT_TIMESTAMP

Tabela reservations

Campo Tipo Descrição Restrições
id INTEGER Chave primária auto-incrementada PRIMARY KEY, AUTOINCREMENT
property_id INTEGER Referência à propriedade FOREIGN KEY REFERENCES properties(id)
guest_name TEXT Nome do hóspede NOT NULL
guest_email TEXT Email do hóspede NOT NULL
check_in_date TEXT Data de check-in (YYYY-MM-DD) NOT NULL
check_out_date TEXT Data de check-out (YYYY-MM-DD) NOT NULL
total_price REAL Preço total da reserva NOT NULL
status TEXT Status da reserva DEFAULT 'pending'
created_at TIMESTAMP Data de criação DEFAULT CURRENT_TIMESTAMP
updated_at TIMESTAMP Data de atualização DEFAULT CURRENT_TIMESTAMP

Relacionamentos

  • reservations.property_id referencia properties.id
  • Relação: Uma propriedade pode ter múltiplas reservas (1:N)
  • Integridade referencial: Não é possível criar reserva para propriedade inexistente

Validação de Dados

O sistema implementa validação usando Joi schemas:

  • Campos obrigatórios são validados na criação/atualização
  • Tipos de dados são verificados (string, number, date)
  • Formato de email é validado
  • Datas são verificadas para formato YYYY-MM-DD
  • Preços devem ser números positivos
