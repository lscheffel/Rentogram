Fluxos de Execução

Esta seção descreve os fluxos de execução para desenvolvimento, inicialização e operação do sistema Rentogram.

Fluxo de Inicialização

flowchart TD A[Usuário executa start.bat] --> B[Verificar Node.js] B --> C[Instalar dependências npm] C --> D[Iniciar backend em background] D --> E[Abrir navegador em localhost:5274] E --> F[Backend disponível em localhost:3000] F --> G[Instalar dependências frontend] G --> H[Iniciar frontend em localhost:5173]

Fluxo de Desenvolvimento

flowchart TD A[Desenvolvedor] --> B[Executar backend: npm run dev] A --> C[Executar frontend: cd src/frontend && npm run dev] B --> D[Backend rodando em localhost:3000] C --> E[Frontend rodando em localhost:5173] D --> F[Testar integrações API] E --> F F --> G[Desenvolver no frontend] G --> H[Testar mudanças] H --> I{Mudanças no backend?} I -->|Sim| J[Reiniciar servidor backend] I -->|Não| K[Continuar desenvolvimento] J --> K

Fluxo de Interação com API

flowchart TD A[Frontend Component] --> B[Enviar requisição HTTP] B --> C[Middleware CORS] C --> D[Middleware Validação] D --> E[Controller apropriado] E --> F[Model] F --> G[Banco de dados SQLite] G --> F F --> E E --> H[Resposta JSON] H --> A

Operações CRUD

Propriedades

  • Criar: POST /api/properties → Validação → Inserção no banco → Retorno do ID
  • Ler: GET /api/properties → Consulta ao banco → Retorno da lista
  • Atualizar: PUT /api/properties/:id → Validação → Update no banco → Retorno dos dados
  • Excluir: DELETE /api/properties/:id → Delete no banco → Confirmação

Reservas

  • Criar: POST /api/reservations → Verificar propriedade existente → Inserção → Retorno
  • Ler: GET /api/reservations → Consulta com joins → Retorno da lista
  • Atualizar: PUT /api/reservations/:id → Validação → Update → Retorno
  • Excluir: DELETE /api/reservations/:id → Delete → Confirmação

Tratamento de Erros

O sistema implementa tratamento de erros consistente:

  • Erros de validação retornam status 400 com mensagens descritivas
  • Recursos não encontrados retornam status 404
  • Erros internos retornam status 500 com logs apropriados
  • Middleware global captura e formata todas as respostas de erro
