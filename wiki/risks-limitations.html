Riscos e Limitações

Esta seção identifica os riscos conhecidos e limitações do sistema Rentogram.

Limitações Conhecidas

  • Banco de dados local: SQLite não é escalável para produção com múltiplos usuários.
  • Sem autenticação: Não há controle de acesso ou autorização implementado.
  • Frontend limitado: Interface básica sem recursos avançados de UX.
  • Testes ausentes: Não há suite de testes automatizados.
  • Validações básicas: Validações de dados podem ser insuficientes para cenários complexos.

Riscos de Segurança

  • SQL Injection: Embora usando prepared statements, há risco se não forem usados corretamente.
  • CORS: Configuração permite qualquer origem em desenvolvimento.
  • Dados sensíveis: Informações de hóspedes não são criptografadas.

Limitações de Performance

  • Escalabilidade: Arquitetura não suporta horizontal scaling.
  • Queries: Consultas N+1 podem ocorrer sem otimizações.
  • Frontend: Sem otimização de bundle ou lazy loading.

Melhorias Futuras

  • Implementar autenticação e autorização
  • Migrar para banco de dados relacional escalável (PostgreSQL)
  • Adicionar testes automatizados
  • Melhorar interface de usuário
  • Implementar cache e otimização de performance
